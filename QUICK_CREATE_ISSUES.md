# イシューを素早く作成する方法

GitHub CLIでネットワークエラーが発生しているため、以下の方法でイシューを作成してください。

## 方法1: GitHub Web UIから直接作成（推奨）

### Issue #1: バグ修正
1. https://github.com/akkiy93/task-portal/issues/new?title=バグ修正: タスク追加・編集ボタンの二重送信問題&body=## 問題%0Aタスク追加ボタンや編集ボタンを押した際に、タイムラグがあり追加/編集できていないと感じ、2回目を押すと2個追加/編集される問題が発生しています。%0A%0A## 再現手順%0A1. タスク追加フォームに入力して「タスクを追加」ボタンをクリック%0A2. すぐに反応がないため、もう一度クリック%0A3. 結果として2つのタスクが追加される%0A%0A同様に編集ボタンでも2回クリックすると編集欄が2つ表示される%0A%0A## 期待される動作%0A- ボタンをクリックしたら即座に無効化（disabled）する%0A- 処理中はローディング表示を出す%0A- 処理完了までボタンを再度クリックできないようにする%0A%0A## 解決方法%0A- ボタンクリック時に即座にdisabled属性を追加%0A- ローディングインジケーターを表示%0A- 処理完了後にボタンを再有効化&labels=bug を開く
2. ラベル `bug` が自動で選択されていることを確認
3. 「Submit new issue」をクリック

### Issue #2: UI改善
1. https://github.com/akkiy93/task-portal/issues/new を開く
2. タイトル: `UI改善: ヘッダー部分の機能追加`
3. 本文: `issue2_ja.md` ファイルの内容をコピー＆ペースト
4. ラベル: `enhancement` を選択
5. 「Submit new issue」をクリック

### Issue #3: 機能追加
1. https://github.com/akkiy93/task-portal/issues/new を開く
2. タイトル: `機能追加: 体調記録欄を日報記入欄に変更`
3. 本文: `issue3_ja.md` ファイルの内容をコピー＆ペースト
4. ラベル: `enhancement` を選択
5. 「Submit new issue」をクリック

## 方法2: ファイルから本文をコピー

リポジトリの以下のファイルから本文をコピーできます：
- `issue1_ja.md` - Issue #1の本文
- `issue2_ja.md` - Issue #2の本文
- `issue3_ja.md` - Issue #3の本文

これらのファイルはGitHubにプッシュ済みなので、リポジトリから直接コピーできます。
