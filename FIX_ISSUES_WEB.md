# イシューの文字化け修正手順（Web UI）

## 現在の状況

- Issue #3: タイトルが文字化け（`UI謾ヶ蝟・綱倥ャ繝繝シ驛ィ蛻・・讖溯・霑蜉`）
- Issue #2: 英語のタイトル（`Feature: Convert health record section to daily report entry`）
- Issue #1: 未作成または表示されていない

## 修正手順

### Issue #3のタイトルを修正

1. https://github.com/akkiy93/task-portal/issues/3 を開く
2. タイトルの右側にある「✏️」アイコンをクリック（またはタイトルを直接クリック）
3. タイトルを以下に変更：
   ```
   UI改善: ヘッダー部分の機能追加
   ```
4. 「Save」ボタンをクリック

### Issue #2のタイトルを日本語に変更

1. https://github.com/akkiy93/task-portal/issues/2 を開く
2. タイトルの右側にある「✏️」アイコンをクリック
3. タイトルを以下に変更：
   ```
   機能追加: 体調記録欄を日報記入欄に変更
   ```
4. 「Save」ボタンをクリック

### Issue #1を作成（未作成の場合）

1. https://github.com/akkiy93/task-portal/issues/new を開く
2. タイトルを以下に入力：
   ```
   バグ修正: タスク追加・編集ボタンの二重送信問題
   ```
3. 本文に以下をコピー＆ペースト（`issue1_ja.md` の内容）：

```markdown
## 問題
タスク追加ボタンや編集ボタンを押した際に、タイムラグがあり追加/編集できていないと感じ、2回目を押すと2個追加/編集される問題が発生しています。

## 再現手順
1. タスク追加フォームに入力して「タスクを追加」ボタンをクリック
2. すぐに反応がないため、もう一度クリック
3. 結果として2つのタスクが追加される

同様に編集ボタンでも2回クリックすると編集欄が2つ表示される

## 期待される動作
- ボタンをクリックしたら即座に無効化（disabled）する
- 処理中はローディング表示を出す
- 処理完了までボタンを再度クリックできないようにする

## 解決方法
- ボタンクリック時に即座にdisabled属性を追加
- ローディングインジケーターを表示
- 処理完了後にボタンを再有効化
```

4. ラベル `bug` を追加
5. 「Submit new issue」をクリック

## 完成後のイシュー一覧

- Issue #1: バグ修正: タスク追加・編集ボタンの二重送信問題（ラベル: `bug`）
- Issue #2: 機能追加: 体調記録欄を日報記入欄に変更（ラベル: `enhancement`）
- Issue #3: UI改善: ヘッダー部分の機能追加（ラベル: `enhancement`）
