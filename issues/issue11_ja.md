# 体調項目の簡素化と備考欄追加

## 概要

ヘッダーの「現在の体調」と日報記入欄の「体調スコア」を、3項目から2項目に簡素化し、体調の備考欄を追加します。

## 変更内容

### 1. 体調項目の簡素化
- **現在**: 「良い」「普通」「良くない」の3項目
- **変更後**: 「良い」「不調あり」の2項目に変更
- **対象箇所**:
  - ヘッダーの「現在の体調」ボタン
  - 日報記入欄の「体調スコア」ボタン

### 2. 体調の備考欄追加
- 体調の備考欄（テキスト入力）を追加
- 例: 頭痛、腹痛などの不調の内容を保存
- **対象箇所**:
  - ヘッダーの「現在の体調」セクション
  - 日報記入欄の「体調スコア」セクション

## 技術的な実装

### バックエンド（Code.gs）

#### 体調項目の変更
- `recordHealthScore()` 関数: 体調スコアを「良い」「不調あり」の2項目に対応
- 体調備考欄の追加: 新しいパラメータ `healthMemo` を追加
- 新しい日報シートの列構造:
  - 「出社時体調」列: 「良い」「不調あり」の2値に対応
  - 「退社時体調」列: 「良い」「不調あり」の2値に対応
  - 「出社時体調備考」列: 新規追加
  - 「退社時体調備考」列: 新規追加

#### 定数の更新
- 日報シートの列インデックス定数を更新:
  - `REPORT_COL_ARRIVAL_HEALTH`: 出社時体調（「良い」「不調あり」）
  - `REPORT_COL_ARRIVAL_HEALTH_MEMO`: 出社時体調備考（新規）
  - `REPORT_COL_DEPARTURE_HEALTH`: 退社時体調（「良い」「不調あり」）
  - `REPORT_COL_DEPARTURE_HEALTH_MEMO`: 退社時体調備考（新規）

### フロントエンド（index.html）

#### ヘッダーの「現在の体調」セクション
- ボタンを「良い」「不調あり」の2つに変更
- 「普通」「良くない」ボタンを削除
- 体調備考欄（テキスト入力）を追加
- `setHealthStatus()` 関数を更新して備考欄の値を送信

#### 日報記入欄の「体調スコア」セクション
- ボタンを「良い」「不調あり」の2つに変更
- 「普通」「良くない」ボタンを削除
- 体調備考欄（テキスト入力）を追加
- `selectReportHealthScore()` 関数を更新して備考欄の値を送信

#### CSS スタイリング
- 体調備考欄のスタイルを追加
- テキスト入力欄のスタイルを定義

## データ構造

### 新しい日報シートの列（体調関連）

1. 日付
2. 目標
3. 出社時体調（「良い」「不調あり」）
4. 出社時体調備考（新規）
5. 退社時体調（「良い」「不調あり」）
6. 退社時体調備考（新規）
...（その他の列は後続のイシューで定義）

## 注意事項

- Issue #10で作成された新しい日報シートを使用します
- 既存の「日報（旧）」シートには影響しません
- 体調スコアの値は「良い」または「不調あり」のみを受け付けます
