# 日報シートの移行: 旧シートの名称変更と新シートの作成

## 概要

データの後方互換性を考慮しないため、既存の日報シートを「日報（旧）」に名称変更し、新しい列構造を持つ「日報」シートを作成します。

## 変更内容

### 1. 既存シートの名称変更
- 現在の「日報」シートを「日報（旧）」に名称変更
- 既存のデータは保持するが、今後は使用しない

### 2. 新しい日報シートの作成
- 新しい「日報」シートを作成
- 新しい列構造で初期化（列構造の詳細は後続のイシューで定義）

## 技術的な実装

### バックエンド（Code.gs）

#### `initializeSpreadsheet()` 関数の更新
1. **既存シートの名称変更**:
   ```javascript
   // 既存の「日報」シートが存在する場合、「日報（旧）」に名称変更
   const oldReportSheet = ss.getSheetByName('日報');
   if (oldReportSheet) {
     oldReportSheet.setName('日報（旧）');
   }
   ```

2. **新しい日報シートの作成**:
   - 新しい「日報」シートを作成
   - ヘッダー行は後続のイシューで定義（現時点では仮の構造で作成可能）

#### 定数の追加
- `SHEET_NAME_REPORT_OLD = '日報（旧）'` を追加（既存シートの参照用、必要に応じて）

## 注意事項

- 既存の日報データは「日報（旧）」シートに保持されます
- 新しい日報シートは、後続のイシュー（Issue #11、#12）で列構造を定義します
- 既存データの移行は不要です（後方互換性を考慮しない）
