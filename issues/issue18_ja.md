# タスク一覧UIの改善

## 概要

タスク一覧のUIを改善し、領域ごとに3分割表示、ドラッグ&ドロップによる領域間移動、モーダルでのタスク登録を実装します。

## 変更内容

### 1. タスク登録UIの変更
- 既存のタスク登録欄を削除
- 小さな「タスクを登録」ボタンを追加
- ボタンクリックでモーダルを表示
- モーダル内でタスクを登録

### 2. タスク一覧の3分割表示
- 横幅を3分割して領域ごとに表示
  - 「今すぐやる」
  - 「スタックする」
  - 「計画的にする」
- 各領域にタスクを表示

### 3. ドラッグ&ドロップ機能
- タスクをドラッグして領域間を移動可能
- 移動時に領域を自動更新

### 4. 完了したタスクの非表示
- ステータスが「完了」のタスクは画面上に表示しない
- フィルターやボタンで表示可能にする

## 技術的な実装

### フロントエンド（index.html）

#### タスク登録モーダル
- モーダルUIの実装
- フォーム項目: タイトル、説明、見積もり時間、領域、提出先、締切日
- 領域のデフォルト: 「今すぐやる」

#### 3分割レイアウト
- CSS GridまたはFlexboxで3分割
- 各領域をカラムとして表示
- レスポンシブ対応

#### ドラッグ&ドロップ
- HTML5 Drag and Drop APIを使用
- ドラッグ開始時にタスクIDを保持
- ドロップ時に領域を更新
- バックエンドに領域の更新を送信

#### 完了タスクのフィルター
- デフォルトで完了タスクを非表示
- 「完了タスクを表示」トグルボタンを追加
- フィルター状態を保持

### バックエンド（Code.gs）

#### 領域更新関数
- `updateTaskArea(taskId, area)`: タスクの領域を更新
- バリデーション: 領域の値が有効か確認

## 注意事項

- ドラッグ&ドロップはモバイルでも動作するよう、タッチイベントにも対応
- 領域間の移動は即座に反映（オプティミスティックUI更新）
